# Monitoring & Audit (CloudTrail & S3 Logging)

## Project Overview
I enabled AWS CloudTrail and explored S3 access logging to implement basic monitoring and auditing across my AWS environment. The goal was to gain visibility into activity within S3, including who accessed resources, what actions were performed, and when they occurred.

This setup demonstrates foundational cloud security practices by providing an audit trail of API activity and object-level access, which is essential for governance, compliance, and troubleshooting in real-world AWS environments.

## Architecture Components

- **AWS CloudTrail** – service used to track and record AWS API activity
- **Amazon S3 Access Logs** – used to capture requests made to S3 buckets
- **Amazon S3 (Log Storage)** – stores log files generated by CloudTrail and S3 logging
- **IAM (Identity and Access Management)** – controls access to logging and audit resources

## Logging and Monitoring Flow

AWS Services Activity → CloudTrail Capture → Log Delivery to S3 Bucket → Audit Review

## CloudTrail Configuration

CloudTrail was enabled to track management events across AWS services, with a focus on S3 activity.

### Configuration Details:
- Trail type: Multi-region trail
- Management events: Enabled
- Storage destination: S3 bucket for log delivery
- Logging: Read and write events enabled

This ensured that all API-level actions within the AWS account were recorded and stored for analysis.

## S3 Access Logging

S3 server access logging was enabled to track requests made directly to S3 buckets.

### Logged Information Includes:
- Request type (GET, PUT, DELETE)
- Source IP address
- Timestamp of access
- Object accessed
- Response status

This provided deeper visibility into object-level activity beyond CloudTrail API logs.

## Testing and Validation

To validate monitoring functionality, multiple actions were performed on the S3 bucket:

- File uploads
- File access (download attempts)
- Object modifications

### Expected Behaviour:
- CloudTrail records API activity
- S3 logs capture access requests
- Logs are delivered to the configured S3 log bucket

### Result:
Activity logs were successfully generated and stored in the designated S3 bucket, confirming that monitoring and auditing were functioning correctly.

## Key Concepts Demonstrated

- Cloud activity monitoring using AWS CloudTrail
- S3 server access logging configuration
- Audit trail creation for AWS resources
- Security visibility and compliance tracking
- Log storage and analysis using Amazon S3
- Differentiation between API-level and object-level logging

## Skills Demonstrated

- AWS CloudTrail setup and configuration
- S3 logging and monitoring implementation
- Cloud security and audit fundamentals
- Log analysis for AWS resources
- IAM awareness for secure logging access

## Key Learning Outcome

This project demonstrated how AWS provides built-in monitoring and auditing capabilities through CloudTrail and S3 logging. By enabling these services, I gained visibility into account activity and learned how cloud environments can be monitored for security, compliance, and operational transparency.
